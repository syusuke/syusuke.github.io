Spring

依赖注入

  1. 接口注入

  2. set注入

  3. 构造注入

    constructor-arg: 表示构造方法注入

    index: 表示参数的个数

    注入方式的选择

    1. 构造注入可以在构建对象的同时一并完成依赖关系主的建立, 这个对象的关系也就准备好了.
    2. 如果在建立的关系过多, 使用构造方式注入,会使得参数太长,
    3. set 要写好关系的名字,方便区分

Bean的4种依赖检查模式

dependency-check : “”

  1. simple: 是指Bean的属性为基本数据类型,字符串,集合进行依赖检查
  2. object: 是指Bean的属性为Object,对Object进行依赖检查
  3. all: 全部属性
  4. none: 不进行依赖检查
